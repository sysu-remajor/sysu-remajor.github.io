# AGENT 工作说明（SYSU-ReMajor）

本文档用于说明当前仓库的维护方式，方便后续人工或 Agent 协作时保持一致。

## 1. 目标与原则

1. 目标：构建中山大学转专业信息的可检索、可追溯、低重复知识库。
2. 原则：官方优先、年份优先、学院优先、经验补充。
3. 口径：涉及规则结论时，始终以教务部和学院当年公告为准。

## 2. 目录结构

1. 内容仓库：`content/`（也是 Obsidian Vault）。
2. 资源目录：`content/assert/`（图片、PDF、Excel 等统一放这里）。
3. 模板目录：`content/template/`。
4. 主通道页面：`content/5月转院系专业/` 与 `content/9月二次遴选/`。
5. 学院页面：`content/学院/`（一学院一页，必要时“总览 + 方向页”拆分）。

## 3. 标签体系（当前已启用）

学院页统一使用简化标签，不使用 `/` 分层：

1. `学部` 标签：如 `信息学部`、`工学部`、`医学部`。
2. `校区` 标签：如 `东校园`、`南校园`、`北校园`、`珠海校区`、`深圳校区`。

每个学院页默认只保留这两类标签（学部 + 校区），不再细分“考核/通道”。

> 说明：该规范仅用于“学院主页面”（如 `学院/XX学院/XX学院.md`）。  
> 历年真题、历年政策等子页面可按专题标签（年份、真题、政策）维护。

同时在 frontmatter 增加结构化字段：

- `faculty: <学部>`
- `campus: <校区>`

## 4. 页面组织约定

1. 导航页负责分流，不重复展开细节。
2. 学院页固定最小结构：
   - 接收计划
   - 申请要求
   - 考核方式
   - 联系方式
   - 社区补充（非官方）
3. 经验页只给“可执行动作”，不替代政策页。
4. 二次遴选与普通转专业分开维护，不混时间线。

## 5. 链接与命名规范

1. 双链优先使用带路径写法，避免歧义：
   - `[[5月转院系专业/转专业流程|转专业流程]]`
   - `[[9月二次遴选/二次遴选|二次遴选]]`
2. 避免重复 `index` 命名；页面名要表达主题。
3. 资源链接统一相对 `assert/`。

## 6. 信息源优先级

1. 教务部/学院官方通知与 PDF。
2. 学院官网公开说明或 FAQ。
3. 社区经验（小红书、公众号、飞书、群聊、共享文档）。

社区经验必须标注“非官方，仅供参考”。

## 7. 更新流程（建议）

1. 先改 `政策与细则` 与目标学院页。
2. 再改 `FAQ`、`经验与社区`、`学院导航` 做互链。
3. 运行构建检查：`npx quartz build`。
4. 检查重复、断链、歧义双链后再提交。

## 8. 当前重点维护方向

1. 计算机与相邻工科（计院、软工、AI、网安、电信、电通、智工、信管）。
2. 已补医学与经管样例（中山医、医学院深圳、岭南）。
3. 后续持续补全其他学院独立页与年度变化对照。

## 9. 新增页面门槛（避免空壳）

1. 仅在“有实质信息”时新增学院页：至少满足以下之一：
   - 当年官方 PDF 可提取到明确条款（计划/条件/考核/联系方式）
   - 共享文档（Excel）有连续年份样本且可形成可执行建议
2. 不再批量新增“仅有学院名、无可用信息”的页面。
3. 若信息不足，优先补充到已有总览页，等待资料充足后再拆独立页。
