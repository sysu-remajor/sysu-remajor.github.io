欢迎你来一起维护这个仓库。无论是改错、补资料还是优化结构，都会让后面的人少走弯路。

> [!note]
> `content/` 本身就是一个 Obsidian 仓库（Vault）。
>
> - 资源目录：`assert/`
> - 模板目录：`template/`
>
> 这两项已经写入仓库内的 Obsidian 配置，打开即用。

> [!tip]
> 等 Obsidian 12.0 发布后，我们会整理一份面向 AGENT 的 `obsidian-cli` 协作文档，进一步规范自动化处理流程。

## 你可以贡献什么

1. 新增或更新院系政策摘要（接收计划、申请条件、考核方式、录取规则）。
2. 补充历年真题与经验（请标注年份和来源类型：官方/回忆/群整理）。
3. 修复错链、死链、错别字、过期信息。
4. 优化信息架构（去重合并、导航补全、双链修复）。

> [!tip]
> 如果你不确定先补哪一页，优先看 [[学院/学院信息完善看板]]，从“部分/待补”的学院项开始。

## 内容规范（请优先遵守）

1. 一页一主题，避免同一信息在多个页面重复展开。
2. 双链优先使用明确路径，避免歧义链接。
3. 文件名使用语义化命名，避免大量同名 `index` 页面。
4. 所有图片和 PDF 统一放在 `content/assert/` 目录。
5. 正文引用资源时使用相对路径。
6. 涉及政策结论时，尽量附官方出处或对应 PDF。
7. 保留“非官方整理，仅供参考，以当年公告为准”的提示。

## 本地开发（仓库根目录）

```bash
npm ci
npx quartz build --serve
```

> [!tip]
> 预览时重点检查三件事：
>
> 1. 链接是否可跳转。
> 2. 页面是否重复讲同一件事。
> 3. 资源文件是否都在 `content/assert/`。

## Pull Request 提交流程

1. Fork 仓库并克隆到本地。
2. 从 `main` 新建分支。
3. 在 `content/` 下编辑或新增页面。
4. 本地预览并完成自检。
5. 提交 commit 并推送到你的 fork。
6. 发起 Pull Request 到主仓库 `main`。

项目仓库：
[sysu-remajor.github.io](https://github.com/sysu-remajor/sysu-remajor.github.io)

> [!tip]
> PR 标题建议用前缀：`docs:` / `fix:` / `feat:` / `refactor:`
>
> 示例：`docs: 补充软件工程学院 2026 转专业实施细则摘要`

> [!note]
> 一个高质量 PR 通常满足：
>
> - 变更范围清晰（做了什么、没做什么）
> - 结论可追溯（有来源或明确“回忆整理”）
> - 可回滚（不把无关文件混进来）

## 提交前最小自检清单

1. 页面是否与现有内容重复；如重复，是否已合并并互链。
2. 双链是否可跳转，是否存在歧义目标。
3. 图片/PDF 是否全部位于 `content/assert/`。
4. 关键结论是否注明“年份 + 来源类型”。
5. 本地构建是否通过，页面是否可正常打开。
